Migration Cheat Sheet: Markdown to ClearNotation

This document shows the 15 most common Markdown constructs and their ClearNotation equivalents. Each section shows the Markdown syntax first, then the ClearNotation equivalent.

1. Headings

# Heading 1
## Heading 2
### Heading 3

ClearNotation headings are the same, but exactly one space after # is required. #Bad is a parse error.

# Heading 1
## Heading 2
### Heading 3

2. Bold / Strong

This is **bold text** in Markdown.

ClearNotation uses +{...} instead of **...**. No delimiter-run ambiguity.

This is +{bold text} in ClearNotation.

3. Italic / Emphasis

This is *italic text* in Markdown.
This is _also italic_ in Markdown.

ClearNotation uses *{...} with explicit braces. No _underscore_ form exists.

This is *{italic text} in ClearNotation.

4. Inline Code

Use `console.log()` to debug.

Inline code spans use backticks in both formats. No change needed.

Use `console.log()` to debug.

5. Links

Visit [the docs](https://example.com/docs).
See [API Reference][api-ref] for details.

[api-ref]: https://example.com/api

ClearNotation uses [label -> url] with a -> separator. No reference-style links exist.

Visit [the docs -> https://example.com/docs].
See [API Reference -> https://example.com/api] for details.

6. Images

![Alt text](image.png)
![Screenshot](screenshot.png "A screenshot")

ClearNotation uses the ::figure directive. The body is the caption.

::figure[src="image.png"]{
Alt text
}

::figure[src="screenshot.png"]{
A screenshot
}

7. Unordered Lists

- First item
- Second item
  - Nested item
- Third item

ClearNotation lists are flat in v0.1. No nesting is supported.

- First item
- Second item
- Third item

8. Ordered Lists

1. First step
2. Second step
3. Third step

Ordered lists work the same way. Authored numbers are preserved.

1. First step
2. Second step
3. Third step

9. Blockquotes

> This is a quote.
> It can span multiple lines.
>
> > Nested quotes work too.

ClearNotation blockquotes are flat. No nesting, no multi-paragraph blocks.

> This is a quote.
> It can span multiple lines.

10. Fenced Code Blocks

```python
def greet(name):
    return f"Hello, {name}"
```

ClearNotation fences look the same but the language tag is mandatory. Omitting it is a parse error.

```python
def greet(name):
    return f"Hello, {name}"

---

## 11. Horizontal Rules

::source[language="markdown"]{
---
}

Horizontal rules are identical. Use `---` on its own line.

```cln
---

12. Tables

| Name   | Role      |
|--------|-----------|
| Alice  | Engineer  |
| Bob    | Designer  |

ClearNotation uses the ::table directive instead of pipe tables. No separator row needed.

::table[header=true, align=["left", "left"]]{
Name | Role
Alice | Engineer
Bob | Designer
}

13. Footnotes / Notes

This needs clarification[^1].

[^1]: Here is the explanation.

ClearNotation uses inline-only notes with ^{...}. No reference-style footnotes.

This needs clarification^{Here is the explanation.}.

14. Callouts / Admonitions

> **Warning:** Do not run this in production.

> [!WARNING]
> Do not run this in production.

ClearNotation uses the ::callout directive with a kind attribute.

::callout[kind="warning", title="Warning"]{
Do not run this in production.
}

15. Not Supported in ClearNotation v0.1

The following Markdown constructs have no equivalent in ClearNotation v0.1:

  • Inline HTML: <div>, <span>, <br> and all other HTML tags are not allowed
  • Reference-style links: [text][id] with [id]: url definitions
  • Reference-style footnotes: [^id] with [^id]: text definitions
  • Autolinks: <https://example.com> auto-linking; use explicit [label -> url] instead
  • Nested lists: indented sub-lists are deferred to a future version
  • Nested blockquotes: > > nested quoting is not supported
  • Multi-paragraph list items: list items are single-line only
  • YAML frontmatter: use ::meta{...} instead
  • Code fences without a language tag: always specify the language

Quick Reference

Bold: +{text} not **text**. Italic: *{text} not *text*. Links: [label -> url] not [label](url). Notes: ^{inline text} not [^ref]. Tables: ::table[header=true]{...} not pipe syntax. Images: ::figure[src="path"]{caption} not ![alt](path).
