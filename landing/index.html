ClearNotation

Technical documentation, clearly.

A docs-first markup language with a normative EBNF grammar, fail-closed parsing, and typed extensibility. Not Markdown-compatible by design.

Try the live editor View on GitHub

Syntax

Clean by design, not by accident

ClearNotation uses explicit, unambiguous syntax. Every construct has one form. The parser either succeeds completely or fails with a precise error.

ClearNotation .cln 
# document metadata is typed and validated
::meta{
title = "API Reference"
authors = ["Alice", "Bob"]
}

# headings always start with #
# Authentication

All requests require a +{Bearer token}.
See [the guide -> /docs/auth] for setup.

# code blocks require a language tag
```http
Authorization: Bearer <token>
```

::callout[kind="warning", title="Rate limits"]{
Requests are limited to 1000 per hour.
}
Markdown .md 
# no document metadata standard
# (YAML frontmatter varies by tool)
---
title: API Reference
authors: [Alice, Bob]
---

# headings are also used for structure
## Authentication

All requests require a **Bearer token**.
See [the guide](/docs/auth) for setup.

# language tag is optional (often omitted)
```
Authorization: Bearer <token>
```

# callouts: tool-specific, no standard
> **Warning:** Rate limits
> Requests are limited to 1000 per hour.

Why ClearNotation

Built for documentation that ships

Markdown accumulates ambiguity. ClearNotation is specified, not improvised.

Normative EBNF grammar

The spec is the grammar. Every valid document maps unambiguously to a typed AST. No implementation-defined behavior.

Fail-closed parsing

Unknown directives, unresolved references, and missing includes all produce errors — not silently degraded output.

Typed extensibility

Custom directives are registered in clearnotation.toml, never inline in documents.

One syntax, one meaning

Strong is +{text}. Links are [label -> url]. No shorthand variants to memorize or misuse.

Multi-implementation

Reference Python pipeline, JavaScript normalizer/renderer, tree-sitter grammar, and VS Code extension — all from one spec.

57 conformance fixtures

The test suite covers valid documents, parse errors, and validation errors. New implementations can verify compliance independently.

Get started

Three ways to use ClearNotation

Pick the integration that fits your workflow. All three share the same spec.

VS Code Extension
Syntax highlighting, validation, and preview
ext install clearnotation.clearnotation
Python CLI
Build, check, format, and watch .cln files
pip install clearnotation
JavaScript
Normalize and render .cln in the browser or Node
npm install clearnotation-js
